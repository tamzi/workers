# System Overview

## Purpose

Describe the system at a high level so new contributors can understand how the parts fit together.

## Scope

State what this system includes and what it explicitly does not include.

## Components

- Primary services, modules, and boundaries.
- External systems and integrations.
- Shared infrastructure and tooling.

## Data Flow

- Key data inputs and outputs.
- Major flows between components.
- Storage and persistence boundaries.

## Interfaces

- Public APIs exposed by the system.
- Internal contracts between modules.

## Dependencies

- Critical dependencies and their owners.
- Version or platform constraints.

## What to Capture

- A plain-language description of what the system does.
- The top-level components and their responsibilities.
- Data sources, destinations, and ownership.
- Trust boundaries and integration points.
- Known risks or scaling considerations.
