# Workflow Rules

> **For AI Agents:** Work planning, review, and delivery rules

## Planning

- Clarify requirements before implementation.
- Identify files and modules that will change.
- Define verification steps up front.

## Execution

- Follow the workflow order in `agents/workflows/`.
- Keep steps atomic and focused.

## Review

- Reviewer does not implement.
- QA validates independently.
- Documentation updates happen after implementation.

## Summary

- Provide a concise summary of what changed and why.
- Call out any risks or follow-up work.
