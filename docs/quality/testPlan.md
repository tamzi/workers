# Test Plan

## Scope

Define what is in scope for testing.

## Test Types

- Unit tests for core logic.
- Integration tests for module boundaries.
- End-to-end tests for critical flows.

## Coverage Targets

Define coverage expectations and thresholds for new work.

## Environments

List supported environments and configurations.

## Exit Criteria

Define what must pass before a change is considered complete.

## What to Capture

- The critical user flows that must be covered.
- The minimum set of tests required for each change.
- Any special environments or fixtures needed.
- Exit criteria for QA and release readiness.
